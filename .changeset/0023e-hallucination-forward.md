Forward two content-free structural-existence counts on `/v1/actuals` so the server can report how often, for tasks like yours, code **runs but references a symbol that doesn't exist**.

- **SDK:** `ActualsRequest` gains two optional additive integers — `externalSymbols?` and `unresolvedSymbols?` — that serialize on the wire as `external_symbols` / `unresolved_symbols` (snake-case-safe). They are **counts, not content**; the server turns them into a coverage-gated, regional rate and drops what it doesn't use, exactly as with `trace`.
- **MCP:** the Claude Code `on-session-end` auto path measures, over the run's **produced `.py` artifacts** (successful `Edit`/`Write`/`MultiEdit` targets still present at session close), two integers with a **linter-grade static resolver** and forwards them on the existing actuals POST:
  - `external_symbols` — distinct **external, top-level** module imports across those files (relative/local imports and the project's own modules are excluded; `os.path` counts once, under `os`).
  - `unresolved_symbols` — of those, how many `importlib.util.find_spec` finds **confidently absent** in the interpreter that produced them. `find_spec` on a top-level name resolves via the path finders **without executing** the module body — no third-party import ever runs. Always `≤ external_symbols`.
- **Counts, not content.** Exactly two integers leave the machine — **no symbol names, no import statements, no file paths, no code**. The resolver reads the produced files **locally** and returns only the two counts; this is a stronger privacy position than the trace's redacted `target` (a symbol resolver *could* leak names, so this one deliberately never emits them). **Measured, never fabricated:** both come from a static resolver over observed artifacts; there is no model-invokable tool that can write them (`estimate` remains the only one). **Static, no arbitrary execution:** produced files are AST-parsed, never run.
- **Conservative — under-count, never over-count.** A symbol is counted `unresolved` **only** when `find_spec` confidently returns `None`. Every ambiguity is treated as resolved: unparseable files are skipped, only **unconditional, module-level** imports are considered (a guarded `try/except ImportError` or a function-local import is never counted), the project's **own** top-level packages/modules are excluded and their package/import roots (walked up through `__init__.py`, so src-layout and nested packages work regardless of cwd) are put on `sys.path` so a local package resolves rather than being miscounted absent, and any resolver error treats the name as resolved. **Fail-closed:** no produced Python, no interpreter, a resolver error/timeout, or malformed output omits **both** counts and submits the total unchanged — the token total is the contract. **Opt-out:** `BUDGETARY_TRACE_TARGET=off` suppresses these counts (the resolver isn't even run), together with the trace descriptor and the change counts.
- **Structural existence only; the client classifies nothing.** This is not semantic correctness and not a per-file "you hallucinated" flag — it's whether a referenced top-level module exists. The server owns the rate and any verdict. **Deferred (named, not built):** deep resolution (submodules, imported members `from X import y`, attribute existence `X.foo`) — that needs importing modules and so has side effects; v1 is top-level imports only. Non-Python ecosystems (TS/JS/Go…) are omitted (Python-first). Claude Code only; Codex and third-party hosts expose no produced-artifact surface, so they forward nothing (never fabricated).
